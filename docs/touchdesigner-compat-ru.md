# TouchDesigner: совместимость протоколов (кратко)

## Что продолжает работать без изменений

1. **Исходящий OSC** на TD: `/treadmill/runSession` — тот же состав аргументов, что и раньше.
2. **Входящий OSC (старый канал):** `/treadmill/ack` с `(runSessionId, free|busy)`.
3. **HTTP:** `POST /api/run-result` и при необходимости токена — `POST /api/touchdesigner/run-result`.

Старые интеграции могут по-прежнему использовать только **ack + HTTP** или только **HTTP** для финиша.

## Что добавлено (новый единый канал)

На **тот же UDP-порт**, что и ack (по умолчанию `TD_OSC_ACK_LOCAL_PORT`, чаще всего **7001** на машине бэкенда), дополнительно обрабатывается адрес:

**`/treadmill/runState`** (переопределяется через `TD_OSC_RUN_STATE_ADDRESS`)

Аргументы OSC:

| Аргумент | Тип | Смысл |
|----------|-----|--------|
| 0 | строка | `runSessionId` |
| 1 | строка | `start` — дорожка приняла бегуна (как успешный free-ack); `busy` — дорожка недоступна; `stop` — забег завершён |
| 2 | число | для `stop`: время результата, сек (`resultTime`) |
| 3 | число | для `stop`: дистанция, м (`distance`) |

- `start` / `busy` участвуют в **рукопожатии после** `/treadmill/runSession` так же, как классический ack.
- `stop` вызывает **тот же серверный путь сохранения результата**, что и HTTP `POST /api/run-result` (лидерборд, следующий в очереди и т.д.). Сессия должна быть в статусе **running**.

Отключить разбор `runState`: `TD_OSC_RUN_STATE_DISABLED=true`.

## Таймаут ожидания ответа TD

После отправки `/treadmill/runSession` бэкенд ждёт **ack** или **runState** start/busy в течение:

- `TD_OSC_ACK_TIMEOUT_MS` (по умолчанию **30000** мс).

Если ответа нет:

- по умолчанию статус считается **`busy`** — участник **остаётся в очереди** (безопасный сценарий «забег недоступен» / интеграция не ответила).
- чтобы сохранить прежнее поведение со статусом **`unknown`** (и дальнейшую логику `ack !== 'busy'` в коде), задайте  
  `TD_OSC_ACK_TIMEOUT_RESOLVES_TO=unknown`.

## Рекомендуемый путь для новой сцены TD

**Предпочтительно:** единый канал **`/treadmill/runState`** — `start` / `busy` после старта и `stop` с метриками при финале (можно **не** дублировать HTTP, если OSC stop достаточен).

**Запасной вариант:** как раньше — **`/treadmill/ack`** + **`POST /api/run-result`** (или защищённый URL с токеном).
