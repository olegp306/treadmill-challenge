# Версионирование (Treadmill Challenge)

## Единый номер версии

- **Источник правды:** корневой `package.json` поле `version` (пакет `treadmill-challenge`).
- Формат: **SemVer** (`MAJOR.MINOR.PATCH`), например `1.1.0`.
- Воркспейсы `apps/*` могут дублировать номер для удобства, но релиз считаем по **корню**.

## Что обновлять при изменениях

1. Поднять `version` в **корневом** `package.json`.
2. Добавить запись в **`CHANGELOG.md`** (что сделано, для кого важно).
3. Собрать фронт (`npm run build:frontend`), чтобы в бандл попала новая **`VITE_APP_VERSION`** (читается из корневого `package.json` в `vite.config.ts`).
4. По желанию: git-тег `v1.1.0` на коммит релиза.

## Где видно версию

| Место | Смысл |
|--------|--------|
| Подвал **админки** | «Версия киоска (сборка)» = бандл фронта; «API» = ответ `GET /api/version` |
| `GET /api/version` | JSON `{ name, version }` с бэкенда (читает корневой `package.json` или `APP_VERSION`) |
| `GET /api/public/settings` | Поле `appVersion` (удобно для отладки без отдельного запроса) |

Переопределение на стенде: переменная окружения **`APP_VERSION`** на бэкенде (если задана, имеет приоритет над чтением `package.json`).
