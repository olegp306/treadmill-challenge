# Версионирование и релизы (Treadmill Challenge)

## Единый номер версии

| Что | Где |
|-----|-----|
| **Источник правды** | Корневой `package.json`, поле **`version`** (пакет **`treadmill-challenge`**) |
| Дубликаты для npm | `apps/frontend`, `apps/backend`, `packages/shared` — тот же номер (обновляет скрипт) |

Формат: **SemVer** `MAJOR.MINOR.PATCH` (например `0.1.0`, `0.2.0`, `1.0.0`).

---

## Когда что повышать (правило)

| Бамп | Когда |
|------|--------|
| **PATCH** | Мелкие исправления, дефекты, текст/копирайт, без нового пользовательского сценария и без ломки API/контрактов. |
| **MINOR** | Новая возможность или заметное улучшение UX/flow; обратная совместимость сохранена для существующих клиентов/киоска. |
| **MAJOR** | Ломающие изменения (контракт API, формат данных, обязательные миграции, несовместимое поведение киоска). |

Математика: **patch** — третья цифра `+1`; **minor** — вторая `+1`, третья `0`; **major** — первая `+1`, остальные `0`.

---

## Порядок выпуска версии (алгоритм)

1. Слить/закоммитить код, который входит в релиз.
2. Выполнить бамп одной командой из корня репозитория:
   - `npm run release:patch` **или** `npm run release:minor` **или** `npm run release:major`  
   (это `node scripts/bump-version.mjs …` — обновляет корень и workspace `package.json`).
3. **Сразу** добавить в **`CHANGELOG.md`** секцию `## [X.Y.Z] - YYYY-MM-DD` и перечислить изменения (Added / Changed / Fixed) — коротко, по делу.
4. Собрать фронт: `npm run build:frontend` — в бандл попадает **`VITE_APP_VERSION`** из корневого `package.json` (`vite.config.ts`).
5. Собрать backend при необходимости: `npm run build:backend`.
6. По желанию: **`git tag -a vX.Y.Z -m "vX.Y.Z"`** на коммите релиза.

**Когда писать changelog:** в шаге 3 сразу после бампа версии, до деплоя; так по номеру версии всегда есть текст «что вошло».

## Где видно версию «на стенде»

| Место | Что показывает |
|--------|----------------|
| Главный экран `/` | Серый **`vX.Y.Z`** справа сверху **всегда** (шрифт в 2× мельче, чем раньше после жеста). **Три тапа по «AMAZING»** → переход на **`/admin`** и ввод PIN полной админки. |
| Админка, подвал | «Версия киоска (сборка)» = то, что было в бандле при сборке. |
| `GET /api/version` | `{ version }` с бэкенда (читает корень или **`APP_VERSION`**). |
| `GET /api/public/settings` | Поле **`appVersion`**. |

**Переопределение на сервере:** переменная **`APP_VERSION`** — если задана, API может показывать её вместо чтения `package.json` (полезно для Docker без пересборки).

**Как понять, что задеплоено на устройстве:** открыть главную — **`v…`** виден сразу справа сверху; сверить с последней строкой в `CHANGELOG.md` и с git-тегом.

---

## Частые вопросы

- **Какая сейчас версия в репо?** → корневой `package.json` → `"version"`.
- **Что вошло в эту версию?** → последний блок в **`CHANGELOG.md`** с тем же номером.
- **Чем отличается от предыдущей?** → сравнить два соседних блока в `CHANGELOG.md`.
